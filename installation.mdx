# Installation Guide

Complete guide for installing and configuring claude-mem with Claude Code.

## System Requirements

<CardGrid cols={2}>
  <Card title="Required" icon="check-circle">
    - Node.js 18.0.0 or higher
    - Claude Code desktop application
    - 100MB free disk space
  </Card>
  <Card title="Recommended" icon="star">
    - Bun runtime (faster)
    - macOS/Linux (better hook support)
    - Git for version control
  </Card>
</CardGrid>

## Installation Methods

<Tabs>
  <Tab title="npm (Recommended)">
    <Steps>
      ### Install globally
      ```bash
      npm install -g claude-mem
      ```

      ### Run installer
      ```bash
      claude-mem install
      ```

      ### Restart Claude Code
      Close and reopen the application
    </Steps>
  </Tab>

  <Tab title="Bun">
    <Steps>
      ### Install with bun
      ```bash
      bun install -g claude-mem
      ```

      ### Run installer
      ```bash
      claude-mem install
      ```

      ### Restart Claude Code
      Close and reopen the application
    </Steps>
  </Tab>

  <Tab title="From Source">
    <Steps>
      ### Clone repository
      ```bash
      git clone https://github.com/thedotmack/claude-mem.git
      cd claude-mem
      ```

      ### Install dependencies
      ```bash
      bun install
      ```

      ### Build and link
      ```bash
      bun run build
      bun link
      ```

      ### Install integration
      ```bash
      claude-mem install
      ```
    </Steps>
  </Tab>
</Tabs>

### Hook Integration

claude-mem integrates with Claude Code through hooks:

- **session-start**: Loads relevant memories when you begin
- **session-end**: Compresses and saves conversation
- **pre-compact**: Handles `/compact` command

### MCP Server Configuration

The installer configures the Chroma MCP server in Claude Code:

```json
{
  "mcpServers": {
    "chromadb-claude-mem": {
      "command": "npx",
      "args": ["@modelcontextprotocol/server-chromadb"]
    }
  }
}
```

## Configuration Options

During installation, you can configure:

<AccordionGroup>
  <Accordion title="Save on Clear">
    Automatically compress memories when using `/clear` command
    ```bash
    claude-mem install --save-on-clear
    ```
  </Accordion>

  <Accordion title="Force Reinstall">
    Overwrites existing installation
    ```bash
    claude-mem install --force
    ```
  </Accordion>

  <Accordion title="Custom Directory">
    Install to a different location
    ```bash
    claude-mem install --dir ~/custom/path
    ```
  </Accordion>
</AccordionGroup>

## Updating claude-mem

To update to the latest version:

```bash
npm update -g claude-mem
claude-mem install --force
```

## Uninstalling

To completely remove claude-mem:

```bash
# Remove hooks and config
claude-mem uninstall

# Uninstall package
npm uninstall -g claude-mem

# Optional: Remove data directory
rm -rf ~/.claude-mem
```

<Warning>
  Removing `~/.claude-mem` will delete all your saved memories. Consider backing up first.
</Warning>

## Troubleshooting Installation

<AccordionGroup>
  <Accordion title="Permission denied errors">
    Use `sudo` on macOS/Linux:
    ```bash
    sudo npm install -g claude-mem
    ```
  </Accordion>

  <Accordion title="Command not found">
    Ensure npm's global bin is in your PATH:
    ```bash
    export PATH="$PATH:$(npm bin -g)"
    ```
  </Accordion>

  <Accordion title="Hooks not triggering">
    1. Restart Claude Code completely
    2. Run `claude-mem install --force`
    3. Check logs: `claude-mem logs`
  </Accordion>

  <Accordion title="MCP server errors">
    Ensure chromadb is accessible:
    ```bash
    npx @modelcontextprotocol/server-chromadb
    ```
  </Accordion>
</AccordionGroup>
