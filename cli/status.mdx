Performs a comprehensive health check of the claude-mem installation, verifying hooks, settings, storage, and runtime environment.

## Synopsis

```bash
claude-mem status
```

## Description

The `status` command provides a detailed report of your claude-mem installation:

1. **Hook Scripts** - Verifies installed hook files
2. **Settings Configuration** - Checks Claude Code hook registration
3. **Compressed Transcripts** - Shows archived content statistics
4. **Runtime Environment** - Validates Node.js/Bun availability
5. **Chroma Storage** - Confirms storage backend status
6. **Installation Summary** - Overall system health

## Status Components

### Hook Scripts

Verifies the presence of three core hook files:

- **pre-compact.js** - Handles memory compression before `/compact`
- **session-start.js** - Loads context at session startup
- **session-end.js** - Saves before `/clear` when save-on-clear is enabled

<Note>
Hook files must exist in `~/.claude-mem/hooks/` and have executable permissions
</Note>

### Settings Configuration

Checks hook registration in Claude Code settings:

- **Global Settings** - `~/.claude/settings.json`
- **Project Settings** - `./.claude/settings.json` (if present)

Each hook type is verified:
- âœ… Registered and active
- âŒ Missing or misconfigured

### Compressed Transcripts

Scans for archived conversation data:

- **Compressed files** - `.jsonl.compressed` format
- **Archive files** - `.jsonl.archive` format
- **Location** - Claude projects directory

### Runtime Environment

Validates required runtime dependencies:

- **Node.js** - Required for claude-mem execution
- **Bun** - Alternative runtime (if available)

<Warning>
At least Node.js is required. Missing runtime will prevent claude-mem from functioning.
</Warning>

### Chroma Storage

Confirms vector database configuration:

- **Backend** - Chroma MCP server
- **Location** - Data directory path
- **Features** - Available search capabilities

## Troubleshooting

### Missing Hook Files

If hook scripts are not found:

```bash
# Reinstall hooks
claude-mem install --force
```

### Settings Not Updated

If hooks aren't registered in Claude Code:

1. Check file permissions
2. Restart Claude Code completely
3. Run `claude-mem install --force`

### No Compressed Files

If no archived transcripts are found:

- Normal for new installations
- Files appear after using `/compact` command
- Check projects directory permissions

### Runtime Issues

If Node.js/Bun are missing:

```bash
# Install Node.js
# Visit: https://nodejs.org

# Or install Bun
curl -fsSL https://bun.sh/install | bash
```
