Display activity logs from claude-mem operations to monitor system behavior and troubleshoot issues.

## Synopsis

```bash
claude-mem logs [options]
```

## Description

Shows recent activity from claude-mem operations:

1. **Most Recent Log** - Displays the latest log file by default
2. **Configurable Output** - Control number of lines shown
3. **Multiple Files** - Access to all available log files
4. **Real-time Following** - Watch logs as they update

## Options

| Option | Description | Default |
|--------|-------------|---------|
| `--tail <number>` | Number of recent lines to show | `20` |
| `--all` | Show information about all log files | `false` |
| `--follow` | Follow logs in real-time | `false` |

## Log Levels

Standard log levels:

### INFO
Normal operational messages:
```
2025-09-15T10:30:15.234Z [INFO] Session start: Loading context
2025-09-15T10:45:22.145Z [INFO] Starting compression for session abc123
```

### WARN
Warning conditions that don't prevent operation:
```
2025-09-15T11:15:30.789Z [WARN] Trash directory size: 1.2MB (cleanup recommended)
2025-09-15T11:20:15.456Z [WARN] Large session detected: 500+ messages
```

### ERROR
Error conditions that may affect functionality:
```
2025-09-15T12:00:45.123Z [ERROR] Failed to write memory to index
2025-09-15T12:15:30.789Z [ERROR] Permission denied accessing settings file
```

## Troubleshooting with Logs

### No Log Files Found

```bash
claude-mem logs
```

**Output:**
```
‚ùå No log files found in ~/.claude-mem/logs/
   Run a compression first to generate logs
```

**Solution:**
1. Use `/compact` in Claude Code to trigger logging
2. Check directory permissions for `~/.claude-mem/logs/`
3. Verify claude-mem installation with `claude-mem status`

### Permission Errors

Permission-related log entries:
```
[ERROR] Permission denied accessing settings file
[ERROR] Failed to create log directory
```

**Solutions:**
```bash
# Fix directory permissions
chmod 755 ~/.claude-mem/
chmod 755 ~/.claude-mem/logs/

# Fix file permissions
chmod 644 ~/.claude-mem/logs/*.log
```

### Compression Failures

Compression-related errors:
```
[ERROR] Failed to compress session: Invalid JSON
[WARN] Large session detected: 500+ messages
[ERROR] Compression timeout after 30 seconds
```

**Solutions:**
- Check session transcript format
- Reduce session size before compression
- Increase timeout settings

### Storage Issues

Storage-related problems:
```
[ERROR] Failed to write memory to index
[WARN] Disk space low: 95% full
[ERROR] Index file corrupted, using backup
```

**Solutions:**
```bash
# Check disk space
df -h ~/.claude-mem/

# Verify index file
head ~/.claude-mem/index.jsonl

# Clean up old archives
claude-mem clean --old
```

## Log Analysis

### Extract Specific Information

Use standard Unix tools:

```bash
# Count compression events today
claude-mem logs --tail 1000 | grep "Compression completed" | wc -l

# Find all errors
claude-mem logs --tail 1000 | grep ERROR

# Check memory creation rate
claude-mem logs --tail 1000 | grep "Saved to index" | tail -10
```

### Performance Monitoring

Track system performance:

```bash
# Compression times
claude-mem logs --tail 1000 | grep "Compression completed"

# Session sizes
claude-mem logs --tail 1000 | grep "messages ‚Üí"

# Error frequency
claude-mem logs --tail 1000 | grep ERROR | wc -l
```

## Configuration

### Log Levels

Control verbosity in `~/.claude-mem/settings.json`:

```json
{
  "logLevel": "info",
  "logRetentionDays": 7,
  "maxLogFileSize": "10MB"
}
```

### Environment Variables

Override log settings:

```bash
# Set log level
export CLAUDE_MEM_LOG_LEVEL=debug

# Custom log directory
export CLAUDE_MEM_LOG_DIR=~/custom/logs

# Run with custom settings
claude-mem logs
```

## Integration

### With Shell Scripts

Monitor in scripts:

```bash
#!/bin/bash

# Check for recent errors
if claude-mem logs --tail 100 | grep -q ERROR; then
    echo "‚ùå Recent errors detected"
    claude-mem logs --tail 10 | grep ERROR
    exit 1
fi

echo "‚úÖ No recent errors"
```

### With Monitoring Systems

Export for external monitoring:

```bash
# Export to JSON format
claude-mem logs --tail 1000 --format json

# Send to log aggregator
claude-mem logs --tail 100 | logger -t claude-mem
```

## Performance

### Large Log Files

For active systems:

```bash
# Check log file sizes

# Rotate logs manually
claude-mem logs --rotate

# Clean old logs
find ~/.claude-mem/logs/ -name "*.log" -mtime +7 -delete
```
